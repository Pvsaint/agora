# Issue 87: Implement DAO-Lite Parameter Control

### DESCRIPTION

> **Decentralization is the end-goal for Agora.** This issue transitions the core platform parameters (fees, token whitelist, admin list) from a single admin address to a "DAO-Lite" model requiring multi-address approval or a simple voting period.

### TASKS

- [ ] **Implement `propose_parameter_change`** for any authorized governor.
- [ ] **Add a `VotingPeriod`** (e.g., 48 hours) where other governors can sign off on the change.
- [ ] **Execute the change automatically** once the signature threshold is met and the time-lock expires.
- [ ] **Emit a `GovernanceActionExecuted` event** for public transparency.

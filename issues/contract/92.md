# Issue 92: Implement SVG On-chain Rendering for Tickets

### DESCRIPTION

> **Purely digital tickets benefit from visual representation directly on-chain.** This issue implements an SVG generation engine within the contract, allowing wallets and explorers to render unique ticket artwork (including QR codes, logos, and seat info) without relying on external web servers.

### TASKS

- [ ] **Implement an `svg_builder` module** that concatenates strings and variables into a valid SVG XML format.
- [ ] **Design a standard 'Agora Ticket' template** with placeholders for `event_name`, `tier_name`, and `guest_address`.
- [ ] **Optimize gas usage** for string concatenation by using `Bytes` or `Symbols` where appropriate to keep rendering costs low.
- [ ] **Create a `token_uri` function** that returns the Base64 encoded SVG data for standard compatibility with NFT viewers.
